Decisions tècniques
Registre de decisions del projecte
HTML/CSS pur sense frameworks
Control total del codi sense Bootstrap ni Tailwind. Millora del domini tècnic personal.
✓ Resolt
actualitzar.py injecta HTML directament (sense JSON ni servidor)
Alternativa al sistema fetch()+JSON que fallava en local per restriccions del navegador (file://). L'script escriu els valors directament als comentaris PLM dels HTML.
✓ Resolt
GitHub Actions per al deploy automàtic
git push → Actions executa actualitzar.py → publica la web. Zero intervencions manuals.
✓ Resolt
PLMs Francès i Web sense Excel — format ric manual
El format ric (sessions detallades, gràfics, anàlisis) s'escapa del que pot gestionar un Excel. S'actualitzen editant l'HTML directament o via el xat de seguiment.
✓ Resolt
Redisseny de l'index: menys CV, més projectes
L'index actual dóna massa pes al CV i poc als projectes. Nova proposta: hero concís, projectes com a element visual central i dominant.
⚠ Per resoldre (S4)
PLMs TFM i Elevador: quin format?
Decidir si seguiran el model Excel+actualitzar.py o el model manual ric com Francès i Web.
⚠ Per decidir (S6)
Anàlisi del projecte
Observacions i punts de millora
✅Web en producció — accessible públicament des de qualsevol dispositiu, URL permanent, deploy automàtic en menys d'1 minut.
✅Pipeline robust — git push és l'única acció necessària per actualitzar tota la web. Tots els errors de workflow estan resolts.
✅PLMs de qualitat creixent — el salt del format Excel genèric al format ric manual és significatiu. El PLM de Francès és un bon model a seguir.
⚠️Index per millorar — CV desactualitzat, sense foto, i la secció de projectes queda massa relegada. És la prioritat clara de la propera sessió.
⚠️PLMs TFM i Elevador buits — existeixen com a pàgines però sense contingut real. No és urgent fins que els projectes avancin, però afecta la impressió general.
📌Propera prioritat — redissenyar l'index per posar els projectes al centre. La bio i el CV poden quedar en una secció secundaria o col·lapsable.
